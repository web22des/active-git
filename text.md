## Visual Studio Code Agents (Предварительный просмотр)

Visual Studio Code Agents — это новое предварительное companion-приложение, которое поставляется вместе с VS Code Insiders и создано для разработки с использованием агентов.

- **Параллельное выполнение задач в нескольких проектах** — Запускайте сессии агентов параллельно в нескольких репозиториях (каждая изолирована в своем рабочем дереве), быстро переключайте контекст (с помощью интерфейса, адаптирующегося к вашему выбору) и вносите правки на основе отзывов людей и агентов.

- **Мониторинг и проверка** — Отслеживайте ход сессии, просматривайте различия в коде (diff), оставляйте отзывы агентам и создавайте pull request'ы, не выходя из приложения.

- **Ваши настройки переносятся** — Пользовательские инструкции, файлы с промптами, пользовательские агенты, MCP-серверы, хуки и плагины — всё работает в приложении Agents, равно как и другие ваши настройки VS Code, например, темы.

- **Без дополнительной установки** — Приложение поставляется вместе с VS Code Insiders. Запустите его из меню «Пуск» или папки «Приложения» в вашей ОС, либо выполните команду «Chat: Open Agents Application» из палитры команд.

Приложение Agents — это быстро развивающийся предварительный просмотр. В настоящее время оно доступно только в VS Code Insiders, и мы ждём ваших отзывов в обсуждениях на GitHub.

## Встроенный браузер

В этом выпуске мы продолжаем улучшать работу со встроенным браузером и его возможности для агентов.

**Улучшения инструментов браузера для агентов**  
_Настройка:_ `workbench.browser.enableChatTools`

- **Улучшенные подписи инструментов**  
  Когда агент вызывает инструмент браузера, вызовы теперь имеют более описательную подпись и ссылку для прямого перехода к нужной вкладке браузера.

    _Было:_ скриншот вызова инструмента с надписью «Clicked element in browser».  
     _Стало:_ скриншот вызова инструмента с надписью «Right-clicked header banner in Test Page» и ссылкой на Test Page.

- **Поддержка долгих скриптов**  
  Инструмент «Run Playwright Code» получил улучшенную поддержку долго выполняющихся скриптов. Скрипты, выполнение которых занимает больше пяти секунд (по умолчанию), теперь возвращают агентам отложенный результат для опроса.

- **Меньше дублирующихся вкладок**  
  Теперь агенты активнее предотвращают повторяющееся открытие вкладок браузера. Если агент пытается открыть новую вкладку, а вкладка с тем же хостом уже открыта, новая вкладка не открывается, если только агент не передаст явный флаг.

- **Масштабирование жестом сжатия/разжатия во встроенном браузере (macOS)**  
  Встроенный браузер теперь поддерживает масштабирование жестом сжатия/разжатия на macOS. Используйте жест на трекпаде, чтобы увеличить содержимое веб-страницы до 3 раз.

    В отличие от стандартного масштабирования браузера (Ctrl+= / Ctrl+-), масштабирование жестом является чисто визуальным и не меняет макет страницы.

## Улучшения терминальных инструментов

Этот выпуск улучшает работу агентов с командами, выполняемыми в фоновом режиме в терминале.

- **Отправка ввода в фоновые терминалы**  
  Раньше фоновые терминалы были доступны только для чтения, и был доступен только инструмент `get_terminal_output`. Это было особенно проблематично, когда активный терминал (foreground) завершался по таймауту и переходил в фоновый режим, так как агент больше не мог с ним взаимодействовать.

    С новым инструментом `send_to_terminal` агент может продолжать взаимодействовать с фоновыми терминалами. Например, если сессия SSH завершилась по таймауту при ожидании запроса пароля, агент всё равно может отправить необходимые данные для завершения соединения.

- **Уведомления о фоновых терминалах (Экспериментально)**  
  _Настройка:_ `chat.tools.terminal.backgroundNotifications`

    Раньше, когда команда в терминале выполнялась в фоновом режиме, агенту приходилось вручную вызывать `get_terminal_output`, чтобы проверить её состояние. Не было способа узнать, когда команда завершится или потребует ввода.

    С новой экспериментальной настройкой `chat.tools.terminal.backgroundNotifications` агент автоматически уведомляется, когда команда в фоновом терминале завершается или требует ввода пользователя. Это также относится к активным терминалам, которые завершились по таймауту и были перемещены в фон. Затем агент может предпринять соответствующие действия, такие как просмотр вывода или отправка ввода через инструмент `send_to_terminal`.

## Устаревшие функции и настройки

**Новые устаревания в этом выпуске**  
Отсутствуют

**Будущие устаревания**  
Режим редактирования (Edit Mode) официально устарел, начиная с версии VS Code 1.110. Пользователи могут временно повторно включить Edit Mode с помощью настройки VS Code `chat.editMode.hidden`. Эта настройка будет поддерживаться до версии 1.125. Начиная с версии 1.125, Edit Mode будет полностью удалён, и его больше нельзя будет включить через настройки.
